# Workbench Loop for C64 Development

## Loop

```text
Question -> Source check -> Small experiment -> Build -> Emulator validation -> Evidence -> Capture Back
```

## Good experiments

- print text on screen;
- write directly to screen memory;
- change border/background color;
- move a sprite;
- read joystick input;
- play one SID tone;
- load a charset;
- package a `.d64`.

## Bad experiments for early sessions

- full game engine before a build loop exists;
- undocumented raster timing claims;
- emulator success claimed without a screenshot or pasted observation;
- hardware-specific workflow without noting transfer device and machine model.
